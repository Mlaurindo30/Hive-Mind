## 1. `scripts/sinapse-write.py` — CLI Standalone de Escrita

**Arquivo novo:** `scripts/sinapse-write.py`

Script CLI que qualquer agente pode invocar como ferramenta externa para
persistir decisões e aprendizados no vault. Importa diretamente do módulo
`sinapse_memory` (via importlib, como os testes).

```python
#!/usr/bin/env python3
"""
Sinapse Agent — CLI standalone de escrita para agentes sem plugin Python.
Uso:
  python3 scripts/sinapse-write.py decision --title "Título" --content "Conteúdo"
  python3 scripts/sinapse-write.py learning --title "Título" --content "Conteúdo"
  python3 scripts/sinapse-write.py query "texto da busca"
  python3 scripts/sinapse-write.py health
  python3 scripts/sinapse-write.py session-end --summary "Resumo da sessão"
"""

import argparse, json, os, sys
from pathlib import Path
import importlib.util

# Load the plugin module
_plugin_path = Path(__file__).parent.parent / "plugins" / "hermes" / "sinapse-memory.py"
spec = importlib.util.spec_from_file_location("sinapse_memory", _plugin_path)
sm = importlib.util.module_from_spec(spec)
sys.modules["sinapse_memory"] = sm
spec.loader.exec_module(sm)


def main():
    parser = argparse.ArgumentParser(description="Sinapse Agent — write CLI")
    sub = parser.add_subparsers(dest="command", required=True)

    # decision
    d = sub.add_parser("decision", help="Save a decision to the vault")
    d.add_argument("--title", required=True, help="Decision title")
    d.add_argument("--content", required=True, help="Decision content")

    # learning
    l = sub.add_parser("learning", help="Save a learning to Patterns.md")
    l.add_argument("--title", required=True, help="Learning title")
    l.add_argument("--content", required=True, help="Learning content")

    # query
    q = sub.add_parser("query", help="Query vault knowledge")
    q.add_argument("text", help="Search query")

    # health
    sub.add_parser("health", help="Health check of all backends")

    # session-end
    se = sub.add_parser("session-end", help="End session and update Current State")
    se.add_argument("--summary", required=True, help="Session summary")

    args = parser.parse_args()

    if args.command == "decision":
        result = sm._save_decision(args.title, args.content)
        print(json.dumps({"saved": result is not None, "path": result}))
    elif args.command == "learning":
        result = sm._save_learning(args.title, args.content)
        print(json.dumps({"saved": result is not None, "path": result}))
    elif args.command == "query":
        result = sm._query_vault_knowledge(args.text)
        print(json.dumps(result or {}, default=str))
    elif args.command == "health":
        result = sm.health_check()
        print(json.dumps(result, default=str))
    elif args.command == "session-end":
        sm._session_decisions = []
        sm._session_learnings = []
        sm._update_current_state([], [], args.summary)
        print(json.dumps({"updated": True}))


if __name__ == "__main__":
    main()
```

## 2. `scripts/sinapse-mcp.py` — MCP Server de Leitura + Escrita

**Arquivo novo:** `scripts/sinapse-mcp.py`

MCP server (stdio) que expõe tools para qualquer agente com suporte MCP
(Claude Code, Kilo Code, Codex CLI, Copilot CLI). Implementa o protocolo
JSON-RPC sobre stdin/stdout.

**Tools expostas:**

| Tool | Parâmetros | Descrição |
|------|-----------|----------|
| `sinapse_query` | `query: str` | Busca unificada em todos os backends |
| `sinapse_save_decision` | `title: str, content: str` | Salva decisão no vault |
| `sinapse_save_learning` | `title: str, content: str` | Salva aprendizado no Patterns.md |
| `sinapse_health` | — | Health check completo |
| `sinapse_session_end` | `summary: str` | Finaliza sessão, atualiza Current State |

**Protocolo:** MCP stdio (JSON-RPC 2.0 sobre stdin/stdout)

```python
#!/usr/bin/env python3
"""
Sinapse Agent — MCP Server (stdio)
Expõe ferramentas de leitura e escrita do vault para qualquer agente MCP.

Uso:
  python3 scripts/sinapse-mcp.py

Config MCP do agente:
  {
    "mcpServers": {
      "sinapse-memory": {
        "command": "python3",
        "args": ["scripts/sinapse-mcp.py"],
        "cwd": "<SINAPSE_HOME>",
        "transport": "stdio"
      }
    }
  }
"""

import json, sys
from pathlib import Path
import importlib.util

_plugin_path = Path(__file__).parent.parent / "plugins" / "hermes" / "sinapse-memory.py"
spec = importlib.util.spec_from_file_location("sinapse_memory", _plugin_path)
sm = importlib.util.module_from_spec(spec)
sys.modules["sinapse_memory"] = sm
spec.loader.exec_module(sm)

TOOLS = [
    {
        "name": "sinapse_query",
        "description": "Search the Sinapse vault across all memory backends (NeuralMemory, claude-mem, Graphify knowledge graph)",
        "inputSchema": {
            "type": "object",
            "properties": {
                "query": {"type": "string", "description": "Search query in natural language"}
            },
            "required": ["query"]
        }
    },
    {
        "name": "sinapse_save_decision",
        "description": "Save a decision to the Obsidian vault (cerebro/work/active/)",
        "inputSchema": {
            "type": "object",
            "properties": {
                "title": {"type": "string", "description": "Decision title"},
                "content": {"type": "string", "description": "Decision content/context"}
            },
            "required": ["title", "content"]
        }
    },
    {
        "name": "sinapse_save_learning",
        "description": "Save a learning/pattern/insight to cerebro/brain/Patterns.md",
        "inputSchema": {
            "type": "object",
            "properties": {
                "title": {"type": "string", "description": "Learning title"},
                "content": {"type": "string", "description": "Learning content"}
            },
            "required": ["title", "content"]
        }
    },
    {
        "name": "sinapse_health",
        "description": "Health check of all Sinapse backends and vault",
        "inputSchema": {"type": "object", "properties": {}}
    },
    {
        "name": "sinapse_session_end",
        "description": "End current session and update brain/Current State.md with session summary",
        "inputSchema": {
            "type": "object",
            "properties": {
                "summary": {"type": "string", "description": "Session summary (max 500 chars)"}
            },
            "required": ["summary"]
        }
    }
]

HANDLERS = {
    "sinapse_query": lambda args: sm._query_vault_knowledge(args["query"]) or {},
    "sinapse_save_decision": lambda args: {"saved": sm._save_decision(args["title"], args["content"]) is not None},
    "sinapse_save_learning": lambda args: {"saved": sm._save_learning(args["title"], args["content"]) is not None},
    "sinapse_health": lambda args: sm.health_check(),
    "sinapse_session_end": lambda args: _session_end(args["summary"]),
}

def _session_end(summary):
    sm._session_decisions = []
    sm._session_learnings = []
    sm._update_current_state([], [], summary)
    return {"updated": True}


def handle_request(req: dict) -> dict:
    method = req.get("method")
    req_id = req.get("id")

    if method == "initialize":
        return {
            "jsonrpc": "2.0", "id": req_id,
            "result": {
                "protocolVersion": "2024-11-05",
                "capabilities": {"tools": {}},
                "serverInfo": {"name": "sinapse-memory", "version": "1.0.0"}
            }
        }
    elif method == "tools/list":
        return {"jsonrpc": "2.0", "id": req_id, "result": {"tools": TOOLS}}
    elif method == "tools/call":
        params = req.get("params", {})
        tool_name = params.get("name", "")
        tool_args = params.get("arguments", {})
        handler = HANDLERS.get(tool_name)
        if handler:
            try:
                result = handler(tool_args)
                return {
                    "jsonrpc": "2.0", "id": req_id,
                    "result": {"content": [{"type": "text", "text": json.dumps(result, default=str)}]}
                }
            except Exception as e:
                return {
                    "jsonrpc": "2.0", "id": req_id,
                    "result": {"content": [{"type": "text", "text": json.dumps({"error": str(e)})}], "isError": True}
                }
        else:
            return {"jsonrpc": "2.0", "id": req_id, "error": {"code": -32601, "message": f"Unknown tool: {tool_name}"}}
    elif method == "notifications/initialized":
        return None  # No response for notifications
    else:
        return {"jsonrpc": "2.0", "id": req_id, "error": {"code": -32601, "message": f"Unknown method: {method}"}}


def main():
    while True:
        try:
            line = sys.stdin.readline()
            if not line:
                break
            req = json.loads(line.strip())
            resp = handle_request(req)
            if resp is not None:
                sys.stdout.write(json.dumps(resp) + "\n")
                sys.stdout.flush()
        except json.JSONDecodeError:
            continue
        except KeyboardInterrupt:
            break


if __name__ == "__main__":
    main()
```

## 8. Expansão do `install.sh` — Lógica Multi-Agente

**Arquivo modificado:** `install.sh`

Nova seção após o passo 9 (plugin Hermes): **Passo 10 — Configurar agentes externos**.

```bash
# =============================================================================
# 10. CONFIGURAÇÃO DE AGENTES EXTERNOS (via MCP + templates)
# =============================================================================
echo -e "${BOLD}[10/10] Configurando agentes externos (MCP)...${NC}"

# Detecta agentes instalados
declare -A AGENT_DETECTORS=(
    ["claude-code"]="claude"
    ["codex"]="codex"
    ["openclaw"]="openclaw"
    ["gemini-cli"]="gemini"
)

# MCP config para sinapse-memory (universal)
MCP_SINAPSE_CONFIG=$(cat << 'MCPEOF'
{
    "mcpServers": {
        "sinapse-memory": {
            "command": "python3",
            "args": ["PROJECT_ROOT_PLACEHOLDER/scripts/sinapse-mcp.py"],
            "cwd": "PROJECT_ROOT_PLACEHOLDER",
            "transport": "stdio",
            "enabled": true,
            "description": "Sinapse Agent — Universal Memory Layer"
        }
    }
}
MCPEOF
)
MCP_SINAPSE_CONFIG="${MCP_SINAPSE_CONFIG//PROJECT_ROOT_PLACEHOLDER/$PROJECT_ROOT}"

for agent in "${!AGENT_DETECTORS[@]}"; do
    bin="${AGENT_DETECTORS[$agent]}"
    if command -v "$bin" &>/dev/null; then
        echo -e "  ${GREEN}✓${NC} $agent detectado"

        case "$agent" in
            claude-code)
                # Adiciona ao .mcp.json do projeto ou ~/.claude/.mcp.json
                MCP_FILE="$HOME/.claude/.mcp.json"
                if [ -f "$MCP_FILE" ]; then
                    # Merge config (simplificado: substitui se já existir sinapse-memory)
                    python3 -c "
import json
with open('$MCP_FILE') as f: cfg = json.load(f)
cfg.setdefault('mcpServers', {})['sinapse-memory'] = json.loads('''$MCP_SINAPSE_CONFIG''')['mcpServers']['sinapse-memory']
with open('$MCP_FILE', 'w') as f: json.dump(cfg, f, indent=2)
"
                else
                    mkdir -p "$(dirname "$MCP_FILE")"
                    echo "$MCP_SINAPSE_CONFIG" > "$MCP_FILE"
                fi
                echo -e "    ${GREEN}✓${NC} MCP config → $MCP_FILE"
                ;;
            codex)
                # Codex CLI: config em ~/.codex/mcp.json
                MCP_FILE="$HOME/.codex/mcp.json"
                mkdir -p "$(dirname "$MCP_FILE")"
                echo "$MCP_SINAPSE_CONFIG" > "$MCP_FILE"
                # Template AGENTS.md no vault
                cp "$PROJECT_ROOT/cerebro/.codex/AGENTS.md" "$VAULT_DIR/.codex/AGENTS.md" 2>/dev/null || true
                echo -e "    ${GREEN}✓${NC} MCP config + template → $MCP_FILE"
                ;;
            openclaw)
                echo -e "    ${YELLOW}⊘${NC} OpenClaw: configure MCP manualmente em ~/.openclaw/openclaw.json"
                ;;
            gemini-cli)
                echo -e "    ${YELLOW}⊘${NC} Gemini CLI: configure MCP manualmente"
                ;;
        esac
    fi
done

## 14. Integração Multi-Agente

O Sinapse Agent funciona como camada de memória universal via 3 métodos:

| Método | Agentes | Como funciona |
|--------|---------|--------------|
| **Plugin nativo** | Hermes | `sinapse-memory.py` registra hooks `pre_prompt_build`, `post_tool_use`, `post_session_end` |
| **MCP server** | Claude Code, Codex CLI, Kilo Code, Copilot CLI, OpenClaw, Gemini CLI, ZooCode | `scripts/sinapse-mcp.py` expõe 5 tools via stdio JSON-RPC |
| **CLI standalone** | Qualquer agente com execução shell | `scripts/sinapse-write.py` aceita subcomandos `decision`, `learning`, `query`, `health`, `session-end` |

### Fluxo MCP (Claude Code, Codex, Kilo Code, etc.)

```
Agente externo → MCP tool sinapse_query(query) → sinapse-mcp.py
    → _query_vault_knowledge() → nmem / claude-mem / graph.json
    → resultado JSON → agente usa no contexto

Agente externo → MCP tool sinapse_save_decision(title, content) → sinapse-mcp.py
    → _save_decision() → vault Obsidian (work/active/)
```

### Matriz de Funcionalidades por Agente

| Funcionalidade | Hermes | Claude Code | Codex | Kilo Code | OpenClaw |
|---------------|--------|-------------|-------|-----------|----------|
| Leitura automática (pre_prompt) | ✅ Hook | ✅ MCP tool | ✅ MCP tool | ✅ MCP tool | ✅ MCP tool |
| Escrita automática (post_tool) | ✅ Hook | ⚠️ Manual (tool) | ⚠️ Manual (tool) | ⚠️ Manual (tool) | ⚠️ Manual (tool) |
| Health check | ✅ | ✅ | ✅ | ✅ | ✅ |
